Wiring & Pin Map

ESP32 DevKitC (38-pin). Two-key safety: the ESP32 commands each actuator, the ATtiny85 arms the pump's 24V power. Both must agree for water to move. Lose either and the pump loses power — no source, fail-dry.

CThe arming chain (fail-dry core)

ESP32 — the command key

Energizes the specific zone FET, sets the two diverter-leg FETs, enables the pump relay per schedule. Emits a heartbeat square wave on GPIO4 whenever firmware is healthy.

ATtiny85 — the arm key

Independent MCU. Trips if the heartbeat stops (~2s) or any run exceeds its own hard max-runtime ceiling (~30 min), measured on its own clock — it never trusts the ESP32's sense of time.

Safety relay — the gate

Normally-open, energize-to-pass. The ATtiny holds it closed only while armed. It feeds the armed 24V to the pump (the water source). ATtiny trips → relay opens → pump dies → no source → no water. The valves (on raw 24V) aren't gated here — they pass nothing with the pump off (DEC-012).

DWiring notes that bite if skipped

Clamp coils + valve FETs

1N4007 (or Schottky) flyback across the pump-relay coil and the safety-relay coil. Each valve FET gets a TVS (SMAJ30A) drain-to-source instead — the valves have an internal bridge rectifier, so a freewheel diode across the valve won't clamp the turn-off transient.

Flow sensor level shift

Power the sensor at 5V, but its pulse output swings to 5V — drop it to 3.3V (divider or the level-shifter module) before GPIO27, or you cook the input over time.

Input-only pins

GPIO 34/35/36/39 have no internal pull-ups. Only 36 (ATtiny trip line) is used now — it needs an external 10k to 3.3V. 34/35/39 were the zone buttons, freed by DEC-019 (phone-only); add a pull-up only if you ever reuse one as an input.

Valves: on/off FET, no H-bridge

Every valve is a 2-wire on/off motorized ball valve (NC zones + NC fert leg + NO clean leg). Energize the FET to actuate; the valve travels ~6–10 s, self-cuts at its limit, then holds with negligible draw. De-energize and a capacitor auto-returns it to rest. One GPIO per valve — no H-bridge, no IN1/IN2, no never-both-high.

Common ground

Tie all buck grounds + 24V return + ESP32 GND together at one star point. Floating logic ground against a switching 24V rail is the classic intermittent-glitch source.

Valve FET gates

Each valve FET (IRLZ44N, logic-level, fine at the valve's ~0.1A with a 3.3V gate) gets a 100Ω series gate resistor + 100k gate-to-GND pulldown so it sits off — valve at rest (NC closed / NO open) — during ESP32 boot.

Valves on raw 24V, pump on armed 24V

The valve FETs run off raw (always-on) 24V; only the pump is on the armed (safety-relay) 24V. The pump is the source gate (DEC-012), so valves don't need gating — they pass nothing with the pump off — and keeping them off the armed rail avoids a de-arm-vs-close timing race. With on/off FETs (1 GPIO each) there are no strapping pins in play, so the old DEC-007 GPIO12/15 brick-boot hazard is gone.
