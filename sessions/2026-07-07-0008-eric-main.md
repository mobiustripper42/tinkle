## What happened

**1. ATtiny BOD fuse burned on all 3 chips (closes the Session-18 finding).**
- Root cause from S18: hfuse left factory `0xDF` = brown-out detection OFF → quick
  repower wedges the chip. Fix = fuse-only `hfuse:w:0xDD:m` (BODLEVEL 2.7V).
- Uno-as-ISP on `/dev/ttyACM0` (this machine has it; dialout OK). Wiring: Uno
  13→7, 12→6, 11→5, 10→1(reset), 5V→8, GND→4, + 10µF across Uno RESET/GND.
- All three: read `lfuse 0xe2` ✓ (8MHz took), `hfuse 0xdf` → burned `0xDD`,
  read back `0xdd` verified. No firmware reflash needed. Signature t85 ✓.

**2. Pump brownout root-caused + confirmed (#119).**
- Symptom: zone start → pump runs an instant, immediately off; safety relay coasts
  ~2s then drops; **no new fault logged**.
- Cause: ESP32's 5V (via 24V→buck) shares the 24V rail with the pump. Pump inrush
  sags 24V → buck sags → ESP32 internal brownout reset. GPIOs drop → pump relay
  opens instantly; heartbeat stops → ATtiny holds ARM for its 2s HB_TIMEOUT then
  drops. Empty fault log = chip reset, not a firmware decision.
- **Confirmed:** ESP32 on a separate USB source (GND common) → pump runs every
  time, no reset. Fix = give the ESP32 its own supply, independent of the pump rail.
  Bulk caps only blunt it (100µF far too small; even 1000µF marginal vs motor inrush).

**3. Zone 3 (pin 16) valve fault — diagnosis in progress (#120).**
- Zones 1,2 + both diverter legs fine. Zone 3 opens weakly / ~30s / often not.
- Valve 3 is GOOD: opens on direct 24V and when driven from channel 1.
- Zone-3 FET switches (continuity drain→GND on run, open when stopped) → not welded.
- Fault follows the CHANNEL: channel 3 opens any valve only weakly. Reduced power.
- Leading cause: partially-damaged zone-3 MOSFET (high R_DS(on), stuck linear) →
  valve under-powered. Secondary: marginal pin-16 gate drive, or high-R joint in
  the channel-3 drain/24V path.
