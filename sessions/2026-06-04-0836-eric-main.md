## Task 2: Fix C++11 build break + align native tier to target standard (unplanned)

**Completed:**
- Triggered by the **first-ever on-target `pio run -e esp32`** (toolchain finally installed this session). It failed compiling `run_controller.cpp`: `RunSummary{...}` aggregate-init is illegal under `-std=gnu++11` (arduino-esp32 2.0.x) because `RunSummary` has default member initializers — aggregate-legal only in C++14+. The native tier had compiled at a newer default, so the bug passed `pio test` but broke the firmware. Pre-existing in merged #11.
- `platformio.ini` — pinned `[env:native]` to `-std=gnu++11` so the host tier compiles `src/core` under the **same** language rules as the ESP32. Tiers can no longer diverge; this bug class now caught in ~1s on the laptop.
- `src/core/run_controller.cpp` — replaced the two `RunSummary{...}` brace-inits with a file-local `makeSummary()` helper (field assignment); C++11-clean, de-duplicated, behavior-preserving.

**Verification:** Reproduced the error on native after pinning gnu++11, then confirmed the fix — `pio test -e native` 28/28 green under gnu++11; `pio run -e esp32` **links `firmware.bin`** (Flash 20.7% / RAM 6.6%) with TM1637 temporarily disabled.
